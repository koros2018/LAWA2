## 🔍 失败原因分析

### 1. 路由未注册/导入错误 (5项) 🔴 高优先级

| 测试项 | 错误 | 原因 |
|--------|------|------|
| 获取语料列表 | 404 Not Found | `src/routes/seed_content.py` 导入 `get_current_user` 失败 |
| 获取系统语料 | 404 Not Found | 同上 |
| 获取日志 | 404 Not Found | `src/routes/logs.py` 导入失败 |
| 日志统计 | 404 Not Found | 同上 |
| 错误统计 | 404 Not Found | `src/routes/errors.py` 导入失败 |
| 获取词汇列表 | 404 Not Found | 路由未正确注册 |
| 获取词汇统计 | 404 Not Found | 同上 |

**根因**: `src/middleware/auth_middleware.py` 中没有导出 `get_current_user` 函数，只有 `AuthMiddleware` 类。

**解决方案**: 在每个路由文件中定义本地 `get_current_user` 依赖函数，简化认证逻辑。

### 2. 参数不匹配 (4项) 🟡 中优先级

| 测试项 | 错误 | 原因 |
|--------|------|------|
| 系统统计 | 422 | Admin 路由使用 `Path(user_id)` 而非 `Query(user_id)` |
| 用户列表 | 422 | 同上 |
| 生成问候 | 422 | `generate-greeting` 需要 `event_id` 参数 |
| 推送测试 | 422 | 需要 `title`, `content` 等请求体字段 |

**解决方案**: 测试脚本使用正确的参数格式，或 API 设计统一使用 Query 参数。

### 3. 用户不存在 (1项) 🟢 低优先级

| 测试项 | 错误 | 原因 |
|--------|------|------|
| 获取用户信息 | 404 | 测试用户 `test_user` 未创建 |

**解决方案**: 测试前创建用户，或使用已存在的用户 ID。

### 4. 方法不匹配 (1项) 🟢 低优先级

| 测试项 | 错误 | 原因 |
|--------|------|------|
| 生成问候 | 405 | 测试用 GET，实际需要 POST |

**解决方案**: 测试脚本使用正确的 HTTP 方法。

### 5. 文件上传 (1项) 🟢 低优先级

| 测试项 | 错误 | 原因 |
|--------|------|------|
| 上传图片 | 422 | 需要 `multipart/form-data` 而非 JSON |

**解决方案**: 跳过此测试，或实现文件上传测试。

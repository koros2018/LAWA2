## 0. 核心洞察回顾

Alice 不是要"学中文"，她要的是**被接纳的信号**。

| 场景 | Alice 的触发 | LAWA2 的机会 |
|------|-------------|-------------|
| 面摊说"多点辣" | 老板笑了 | → 预演这个场景 |
| 看懂本地朋友的 meme | 能参与笑点 | → 推送今日网络用语 |
| 群聊回复中文 | 没人迁就她 | → 自然社交用语输入 |

**内部触发（Internal Trigger）：**
- "我本来可以懂，但没有" → 自我失望 → 打开 LAWA2
- "他们都在笑什么？" → 社交好奇心 → 打开 LAWA2
- "如果下次我可以说那句话..." → 预演期待 → 打开 LAWA2

## 1. 社交场景语料体系（新内容层）

当前设计只有"信息流"（资讯卡片），需要增加**社交场景层**：

### 1.1 三层内容金字塔

```
      🎯 社交场景（新增）
     ┌─────────────────┐
     │  真实社交用语    │  ← Alice 最需要但现有系统缺失的
     │  网络用语/meme   │
     │  日常对话模板    │
     └─────────────────┘
             ↑
      📰 信息流（现有）
     ┌─────────────────┐
     │  资讯/趣闻       │  ← 可理解输入的基础
     │  文化知识        │
     │  短篇文章        │
     └─────────────────┘
             ↑
      📖 基础能力（现有）
     ┌─────────────────┐
     │  词汇/语法       │  ← 底层支撑
     │  语音/书写       │
     └─────────────────┘
```

### 1.2 社交场景内容类型

| 场景类型 | 内容示例 | 用户感知 |
|---------|---------|----------|
| **🌐 网络用语** | "绝绝子""栓Q""破防了""YYDS" | "原来他们说的是这个意思！" |
| **💬 群聊场景** | "哈哈哈哈笑死""真的吗？""我也是""6" | "下次我也可以这么回" |
| **🍜 生活场景** | "多点辣""打包""多少钱""好吃" | "落地就能用" |
| **😂 梗/表情包** | 热点梗的来龙去脉 | "终于看懂这个表情包了" |
| **🎬 剧综台词** | 当下热门剧的金句 | "啊，这句话原来是这个意思" |

### 1.3 推送策略调整

```
当前（v3）：
  早：资讯 → 午：趣闻 → 晚：回顾

增强（v4）：
  早 🌅：今日网络热词（1个词 + 场景例句）
     "早安！今天的热词是'绝绝子'——表示'太绝了、太好了'"
     配场景：朋友发美食照片 → 评论"绝绝子！"
   
  午 🌤：社交场景练习（1个场景 + 互动）
     "午间挑战：面摊老板问你'要不要辣'，你该怎么说？"
     可选回答：A. 要一点 / B. 多点辣 / C. 不要
     选完 → 系统教正确说法 + 发音示范
   
  晚 🌙：今日社交回顾
     "今天学了3个网络用语：绝绝子/YYDS/破防了"
     "如果你今天在群里发了一条中文回复，会是什么？"
```

## 2. 社交预演引擎（新能力模块）

核心思路：**在安全环境中预演社交场景**，降低真实场景的焦虑。

### 2.1 场景卡片（卡片式预演）

```
┌─────────────────────────────────────┐
│  🍜 场景卡片 · 面馆点餐             │
│                                     │
│  "你要不要辣？"                      │
│                                     │
│  ┌─────┐  ┌─────┐  ┌─────┐        │
│  │ 不要 │  │ 微辣 │  │ 多辣 │        │
│  └─────┘  └─────┘  └─────┘        │
│                                     │
│  点击一个回答 → 听标准发音 → 跟读   │
│                                     │
│  📌 下次遇到：面摊、火锅店、麻辣烫  │
└─────────────────────────────────────┘
```

### 2.2 社交预演链

将零散场景串联为"叙事链"：

```
社交链：去朋友家吃饭
  场景1："你到了吗？" → 回答"到楼下了"
  场景2："带点水果？" → 回答"不用客气"
  场景3："好吃吗？" → 回答"太好吃了！"
  场景4："再来一碗？" → 回答"够了够了谢谢"

用户感知：不是在学习句子，是在预演一顿饭的全流程
```

### 2.3 社交理解度评分

不仅仅是"这个词懂不懂"，而是：

```
社交理解度（新维度）：
  🟢 能看懂 — 明白这个词在社交场景中的意思
  🟡 能使用 — 知道什么时候用、怎么用
  🔴 能创作 — 能在自己的社交互动中自然使用

举例："破防了"
  🟢 理解：知道是"被戳中笑点/泪点"
  🟡 使用：在群聊里看到搞笑内容 → 知道可以回"破防了😂"
  🔴 创作：自己发了一条搞笑内容，配文"笑死，破防了"
```

## 3. 社交圈融入系统（社交层能力）

### 3.1 兴趣圈增强（原 Circles）

现有"兴趣圈"只是标签分类，需要增强为**真实社交预演场**：

```
圈类型示例：
  🍳 美食圈 — 预演点餐/推荐菜品/评价口味
  ✈️ 旅行圈 — 预演问路/订酒店/买票
  🎮 游戏圈 — 预演组队/吐槽/讨论策略
  🎬 追剧圈 — 预演讨论剧情/刷弹幕
  🐱 萌宠圈 — 预演晒猫/夸别人宠物

每个兴趣圈 = 一个"微社交环境"：
  - 圈内推送的内容 = 这个圈子的真实社交用语
  - 圈内互动 = 用目标语言参与讨论（AI 陪练）
  - 圈内排行 = "你今天在这个圈子里说了X句话"
```

### 3.2 社交风险分级

不同场景的社交风险不同，系统自适应难度：

```
风险等级 | 场景示例 | 系统策略
---------|---------|----------
🟢 低风险 | 评论/点赞/发emoji | 允许自由发挥，不纠正
🟡 中风险 | 群聊回复/简短对话 | AI 给出模板，鼓励模仿
🔴 高风险 | 面对面交流/电话 | 预演训练，降低焦虑

自适应：
  用户社交理解度低 → 推送更多低风险场景
  用户社交理解度高 → 引入中/高风险预演
```

### 3.3 "破冰卡"功能

当用户即将进入真实社交场景时，系统提前推送"破冰卡"：

```
📍 场景触发（用户标记/地理围栏）：
  用户标记"明天要去火锅店" → 系统推送：
  
  🃏 火锅店破冰卡
  ┌─────────────────────────────┐
  │ 🍲 必用句：                   │
  │ "我要一个鸳鸯锅"              │
  │ "调料在哪？"                  │
  │ "这个涮几秒？"                │
  │                              │
  │ 💡 如果听不懂老板说什么：      │
  │ "不好意思，能再说一遍吗？"    │
  └─────────────────────────────┘
```

| **Internal Trigger** | "我想看懂这篇" | **"他们都在笑什么？""下次我想说那句话"** |
| **Social Trigger** | （无） | **看到朋友发中文朋友圈 → 想看懂 → 打开 LAWA2** |
| **Anticipatory Trigger** | （无） | **"明天要去火锅店，先看看怎么说"** |

### 4.2 社交可变奖励

| 奖励类型 | 原设计 | 增强设计 |
|---------|--------|---------|
| 低可变 | 成长积分 | 同左 + **社交理解度进度条** |

| 高可变 | 稀有表达 | **"你学会了今天最火的网络用语！"** |
| 社交奖励 | （无） | **"这个场景你已经预演了3次，明天去火锅店试试？"** |

### 4.3 社交投资

| 投资类型 | 原设计 | 增强设计 |
|---------|--------|---------|
| 词汇收藏 | 生词本 | **社交用语收藏 + 场景标签** |
| 句子摘录 | 好句子 | **"我能在社交中使用的句子"集合** |
| 社交记录 | （无） | **预演过的场景记录 + 自信度曲线** |
| 破冰卡历史 | （无） | **去过的地方 + 用过的句子 + 下次可用** |

## 5. AI 能力增强

### 5.1 社交场景感知 AI

```
AI 能力：理解用户"即将进入"的社交场景

输入：
  - 用户标记的场景（"周末要去火锅店"）
  - 地理信息（如果允许）
  - 时间模式（"每周五晚上用户都会出门"）

输出：
  - 相关场景的破冰卡
  - 该场景常用语TOP5
  - 可能遇到的困难预测
  - "如果听不懂，可以说..."

沉默策略：
  - 用户标记场景后 → 主动推送（非沉默）
  - 用户未标记但有规律 → 在合适时间点推送
  - 用户从未标记且无规律 → 不打扰
```

### 5.2 社交自信度评估

```
评估维度（新）：
  1. 场景覆盖度：预演过多少种社交场景
  2. 场景深度：每个场景的预演次数
  3. 社交理解度：社交用语的理解正确率
  4. 社交使用度：在预演中主动输出的比例
  5. 自信度趋势：随时间的变化曲线

呈现方式：
  🗺️ 社交地图
  用户已覆盖的场景用点亮色标注
  未覆盖的场景灰色，点击可查看"需要什么才能解锁"
```

### 5.3 沉默观察者的社交升级

```
原设计：AI 知道何时不介入
增强：AI 知道何时鼓励"真实尝试"

判断逻辑：
  用户完成3次以上社交预演（同场景）→ AI 鼓励真实尝试
  "你已经在这个场景预演了3次，下次试试真实的？"
  
  用户真实尝试后回来 → AI 跟进
  "昨天去火锅店了吗？老板说了什么你没想到的？"
  → 记录新内容 → 更新该场景的语料库
  
  用户真实尝试后没回来 → AI 不追问（尊重隐私）
```

## 7. Alice 的 Day 1 用户旅程（增强版）

```
07:30 🌅 晨间推送
  "早安！今天的热词是'破防了'——网络用语，
   表示被戳中笑点/泪点/槽点。"
  配场景：群里看到搞笑视频 → 可以回"破防了😂"
  → 用户：啊，原来昨天朋友说的是这个意思！
  → 收藏词汇 +1 | 耗时 15秒

12:30 🌤 午间场景
  "午间挑战：你在一家火锅店，服务员问你
   '锅底要什么辣度？'"
  可选：A. 微辣 B. 中辣 C. 变态辣
  用户选了A
  系统：'"微辣"是大部分外国人能接受的辣度，好选择！'
  系统：'要听标准发音吗？' → 跟读
  → 耗时 45秒

21:00 🌙 晚间回顾
  "今日社交回顾："
  - 学了1个网络热词：破防了
  - 预演了1个社交场景：火锅店点锅底
  - 你的社交理解度：🟡 已使用（跟读）

  "明天预告：火锅店进阶——怎么跟老板说'再加一份毛肚'"
  → 用户感知："我在养我的中文社交能力"
  → 耗时 20秒

Day 1 总计：3次打开，约 80 秒
用户感知："我没学中文，我在预演明天的火锅局"
```

## 9. 与当前 MVP 的关系

```
当前 MVP（S1-S2 已交付）：
  ✅ Action Engine（核心循环）
  ✅ Reward Engine（基础可变奖励）
  ✅ Trigger Engine + 信息流
  ✅ 登录 + 画像流程

本次设计（S7 + S4增强）：
  🆕 社交场景内容类型（插入 S2 信息流）
  🆕 社交理解度评估（S4 增强）
  🆕 场景预演引擎（S7）
  🆕 破冰卡功能（S7）
  🆕 社交地图可视化（S7）

建议：S3 之后先做社交内容类型（轻量改动），再做 S7（重型）
  因为社交内容不需要新引擎，只需要新的内容模板 + 推送策略
```
